# データ担当（Data Engineer / Game Balance）

この担当は **ゲームの数値設計とデータ処理コード** を担当する。\
ゲームの面白さや成長速度を決める重要な役割。

------------------------------------------------------------------------

# 役割概要

データ担当は以下を担当する。

-   ゲームバランス設計\
-   ゲームデータの作成（CSV / JSON）\
-   データ読み込みコード作成\
-   判定関数の作成\
-   ゲームロジック担当へのデータ提供

プログラムも一部書くが、**ゲーム全体の流れはロジック担当が担当**する。

# 作成するコード

データ担当は **データを扱う関数** を作成する。

## 歩数から経験値を取得

``` python
def get_exp_from_steps(steps):
    for reward in step_rewards:
        if reward["min"] <= steps <= reward["max"]:
            return reward["exp"]
    return 0
```

## 経験値からレベルを判定

``` python
def get_level_from_exp(exp):
    level = 1
    for lv, required_exp in level_table.items():
        if exp >= required_exp:
            level = lv
    return level
```

## 歩数から状態を取得

``` python
def get_state_from_steps(steps):
    for row in state_table:
        if row["min"] <= steps <= row["max"]:
            return row["state"]
    return "ふつう"
```

# ゲームロジック担当との違い

  担当           役割
  -------------- ----------------------------
  データ担当     ゲーム数値設計、データ管理
  ロジック担当   ゲームの処理の流れを作る

例

**データ担当**

-   歩数 → 経験値\
-   経験値 → レベル\
-   歩数 → 状態

**ロジック担当**

-   歩数取得\
-   経験値追加\
-   レベル更新\
-   UI更新

------------------------------------------------------------------------

# データ担当の重要な仕事

ゲームを面白くするために

-   成長スピード調整\
-   歩数報酬バランス\
-   継続しやすい設計

を調整する。

------------------------------------------------------------------------

# 目標

以下の状態を目指す。

-   1日で少し成長を感じる\
-   1週間でレベルが上がる\
-   1ヶ月続けると大きく成長する
