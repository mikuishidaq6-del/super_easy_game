# 歩数育成ゲームプロジェクト

このプロジェクトでは、**歩数データを利用した育成ゲームアプリ**を開発する。

ユーザーの歩数やセルフケア行動に応じてキャラクターが成長し、  
日々の活動を楽しく継続できることを目的とする。

---

# プロジェクトの特徴

- 歩数データを利用した育成システム
- キャラクターの状態変化
- 個人に合わせた歩数目標の調整
- 日常行動のゲーミフィケーション

---

# システム構成

```
スマートフォン（歩数データ）
          ↓
端末・統合担当
          ↓
ゲームロジック
          ↓
ゲームデータ
          ↓
UI表示
```

---

# 役割分担

このプロジェクトは **4つの役割**で開発を行う。

- UI担当
- ゲームロジック担当
- データ担当
- 端末・統合担当

---

# UI担当（User Interface）

ユーザーが直接触れる **画面・見た目・操作体験** を担当する。

主な役割

- ゲーム画面デザイン
- キャラクター表示
- 歩数・レベルなどの情報表示
- ボタンやメニューの配置
- レベルアップや報酬の演出

ユーザーがゲームを体験する入口となる部分を作る。

---

# ゲームロジック担当（Game Logic）

ゲームの **ルールや計算処理** を担当する。

主な役割

- 経験値計算
- レベルアップ処理
- 報酬計算
- 歩数目標の調整
- 行動データの評価

ゲームの面白さを決めるアルゴリズムを設計する。

---

# データ担当（Game Data）

ゲームの **数値設計と設定データ** を担当する。

主な役割

- レベルテーブル作成
- 報酬テーブル作成
- キャラクター状態テーブル作成
- ゲームバランス調整
- データ管理（CSV / JSON）

ゲームの成長速度や難易度を設計する。

---

# 端末・統合担当（Device / Integration）

スマートフォンの機能を扱い、各機能を **1つのアプリとして統合**する。

主な役割

- 歩数データ取得
- データ保存と読み込み
- UI・ロジック・データの接続
- アプリ起動処理
- 動作確認・デバッグ

アプリを実際に動く状態にまとめる役割。

# 開発の流れ

1. UI担当が画面を作成
2. データ担当がゲーム数値を設計
3. ロジック担当がゲーム処理を実装
4. 端末・統合担当が全機能を接続
5. 動作確認と調整

---

# プロジェクトの目標

- 毎日アプリを開きたくなるゲーム体験
- 健康行動を自然に継続できる仕組み
- シンプルで使いやすいデザイン
