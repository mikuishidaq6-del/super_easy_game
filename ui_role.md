# UI担当（UI / Frontend Designer）

UI担当は、ユーザーが直接触れる **画面・見た目・操作体験**
を設計・実装する役割を担う。\
ゲームの第一印象や使いやすさはUIによって決まるため、非常に重要な担当である。

UI担当は主に以下を担当する。

-   画面デザイン
-   キャラクター表示
-   ボタンやメニューの配置
-   情報表示（歩数・レベルなど）
-   ユーザー操作の実装

------------------------------------------------------------------------

# 役割概要

UI担当の主な役割

-   ゲーム画面のデザイン
-   キャラクター表示の実装
-   操作ボタンの配置
-   情報の見やすい表示
-   ユーザー操作の実装

ユーザーは
**UIを通してゲームを体験する**ため、UI担当はユーザー体験を設計する役割でもある。

------------------------------------------------------------------------

# 1. ゲーム画面の設計

UI担当はゲーム画面のレイアウトを設計する。

例

-   ホーム画面
-   キャラクター画面
-   結果画面
-   設定画面

例（ホーム画面）

    ┌─────────────────┐
     キャラクター表示
          もちねこ

     Lv.3

     今日の歩数
     5234歩

     [記録する]
     [結果を見る]
    └─────────────────┘

------------------------------------------------------------------------

# 2. キャラクター表示

キャラクターの状態に応じて見た目を変える。

例

  状態       表示
  ---------- ----------------
  ねむい     眠そうな表情
  ふつう     通常表情
  元気       元気な表情
  ごきげん   喜んでいる表情

UI担当は

-   キャラクター画像表示
-   アニメーション
-   状態ごとの見た目

を担当する。

------------------------------------------------------------------------

# 3. 情報表示

ユーザーが知りたい情報を見やすく表示する。

例

表示する情報

-   今日の歩数
-   レベル
-   経験値
-   コイン数
-   連続ログイン日数

例

    Lv.5
    EXP 320 / 500

    今日の歩数
    6240歩

------------------------------------------------------------------------

# 4. ボタンと操作

ユーザーがゲームを操作できるようにする。

例

-   記録ボタン
-   結果確認ボタン
-   メニュー
-   戻るボタン

UI担当は

-   ボタンの配置
-   押したときの動作
-   タップ領域

などを設計する。

------------------------------------------------------------------------

# 5. 状態変化の表現

ゲームの進行に応じて画面を変化させる。

例

-   レベルアップ演出
-   報酬表示
-   キャラクター表情変化

例

    🎉 レベルアップ！

    Lv.4 → Lv.5

------------------------------------------------------------------------

# 他担当との違い

## ゲームロジック担当

ゲームロジック担当は **ゲームの計算やルール** を作る。

例

-   経験値計算
-   レベルアップ判定
-   報酬計算

UI担当は **その結果を画面に表示する**。

------------------------------------------------------------------------

## データ担当

データ担当は **ゲームの数値設定** を作る。

例

-   レベルテーブル
-   報酬テーブル
-   状態テーブル

UI担当は **そのデータを画面に表示する**。

------------------------------------------------------------------------

## 端末・統合担当

端末・統合担当は

-   歩数取得
-   データ保存
-   各機能の接続

を担当する。

UI担当は **ユーザーが見る画面部分**を担当する。

# UI担当の目標

UI担当の目標は

-   見やすい画面
-   直感的な操作
-   キャラクターの魅力

を実現することである。

ユーザーが **毎日アプリを開きたくなる体験** を作ることが重要である。
