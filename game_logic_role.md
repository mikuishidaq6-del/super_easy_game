# ゲームロジック担当（強化版）

## 役割

キャラクターの成長や報酬を管理し、ユーザーの行動に応じてゲーム体験を調整する。\
特に
**歩数データと体調データをもとに、個人に合わせた目標と報酬を動的に変更する仕組み**
を設計・実装する。

------------------------------------------------------------------------

# 基本ロジック

ゲームロジック担当はまず **基本の育成システム** を実装する。

## 担当機能

-   経験値（EXP）計算\
-   レベルアップ\
-   コイン管理\
-   連続ログインボーナス\
-   15コインによるログイン復旧

例:

    フェイス入力 → EXP +10
    歯磨き → EXP +3
    服薬 → EXP +2
    100歩 → コイン +1

------------------------------------------------------------------------

# 歩数の動的ロジック（このアプリの特徴）

固定ルールではなく、**最近の歩数データから目標を自動調整**する。

目的:

-   体調が悪い日でも達成感を得られる
-   個人差に対応する
-   無理な運動を防ぐ

------------------------------------------------------------------------

# 歩数目標の決め方

まず **直近数日の平均歩数** を計算する。

    baseline = 直近5日平均歩数

その後、体調によって目標を変更する。

  体調   今日の目標
  ------ ----------------
  良い   baseline × 0.8
  普通   baseline × 0.6
  悪い   baseline × 0.3

------------------------------------------------------------------------

# 報酬の計算

達成率を計算する。

    達成率 = 今日の歩数 ÷ 今日の目標

  達成率     報酬
  ---------- ----------
  50%以上    小報酬
  100%以上   通常報酬
  120%以上   ボーナス

------------------------------------------------------------------------

# 頑張りすぎ検知

直近の歩数が急増した場合、無理をしている可能性がある。

例

    3日平均の150%以上

その場合

    今日は休息モード

------------------------------------------------------------------------

# 将来の拡張（機械学習）

将来的には、機械学習を用いて **最適な目標歩数を予測**することも可能。

入力データ

-   直近歩数
-   フェイススケール
-   ログイン継続日数
-   セルフケア実施状況

出力

    今日のおすすめ歩数
