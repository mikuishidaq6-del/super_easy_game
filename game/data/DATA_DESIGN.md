# データ設計資料

このファイルはデータ担当が作成したCSVファイルとモデルファイルの設計資料です。

---

## CSVファイル一覧

CSVファイルは `data/` フォルダに格納されています。

---

### level_table.csv

**役割：経験値からレベルを判定するテーブル**

| 列名 | 説明 |
|---|---|
| level | レベル（1〜100） |
| required_exp | 前のレベルから必要なEXP |
| total_exp | レベル到達に必要な累計EXP |

**設計方針**
- L1〜L10はREADMEの値をそのまま使用
- L11以降は指数カーブで増加（チームメンバーのdesmosグラフに合わせた）
- L100の必要EXPは25,000
- 1日約90EXP獲得できる想定で、長期入院でも天井に当たりにくい設計

---

### step_rewards.csv

**役割：歩数から獲得EXPを決定するテーブル**

| 列名 | 説明 |
|---|---|
| min_steps | 歩数の最小値 |
| max_steps | 歩数の最大値 |
| exp_reward | 獲得EXP |

| 歩数 | 獲得EXP |
|---|---|
| 0〜99歩 | 5 |
| 100〜299歩 | 15 |
| 300〜599歩 | 30 |
| 600〜999歩 | 50 |
| 1000〜2999歩 | 70 |
| 3000〜5999歩 | 90 |
| 6000〜9999歩 | 120 |
| 10000歩〜 | 150 |

**設計方針**
- 入院中の血液がん患者を対象としているため低歩数の閾値を細かく設定
- 隔離病棟では1日100〜600歩が現実的なライン
- 元気な日は8000歩程度歩ける場合もあるため高歩数帯も設計
- 歩数はメインではなくボーナス的な位置づけ

---

### character_state.csv

**役割：歩数からキャラクターの状態を決定するテーブル**

| 列名 | 説明 |
|---|---|
| min_steps | 歩数の最小値 |
| max_steps | 歩数の最大値 |
| state | キャラクターの状態 |

| 歩数 | 状態 |
|---|---|
| 0〜999歩 | ねむい |
| 1000〜2999歩 | ふつう |
| 3000〜5999歩 | 元気 |
| 6000歩〜 | ごきげん |

**設計方針**
- 元気な日は8000歩程度歩ける場合もあることを考慮して閾値を設定
- 6000歩でごきげんになれる設計
- 体調の良し悪しで状態が変わりやすい幅広い設計

---

### action_rewards.csv

**役割：日常行動ごとの獲得EXPを定義するテーブル**

| 列名 | 説明 |
|---|---|
| action | 行動の種類 |
| exp_reward | 獲得EXP |
| max_count | 1日の最大実施回数 |

| action | exp_reward | max_count | 説明 |
|---|---|---|---|
| face_input | 20 | 1 | フェイス入力 |
| tooth_brushing | 10 | 1 | 歯磨き |
| gargle | 5 | 5 | うがい（最大5回） |
| body_care | 10 | 1 | 体拭き |
| shower | 15 | 1 | シャワー |
| medication | 15 | 3 | 服薬（朝昼晩の最大3回） |

**設計方針**
- フェイス入力はどの体調でも同じEXP（来ただけでOKの精神）
- 全部やった日の最大EXP：20+10+25+10+15+45 = 125 EXP
- 最低限（フェイス入力のみ）でも20 EXP獲得できる
- うがいは最大5回、服薬は朝昼晩の最大3回カウント可能
