# 端末・統合担当（Device / Integration Engineer）

端末・統合担当は、スマートフォンの機能を利用しながら、各担当が作成した機能を1つのアプリとして統合する役割を担う。

この担当は、以下の2つの役割を持つ。

-   **端末機能担当**（スマートフォンの機能を扱う）
-   **統合担当**（各担当の機能を接続する）

UI担当、ゲームロジック担当、データ担当が作った機能をまとめて、**アプリとして実際に動く状態にする役割**である。

------------------------------------------------------------------------

# 役割概要

端末・統合担当の主な役割は以下の通り。

-   スマートフォンから歩数データを取得する
-   アプリ内データの保存・読み込みを行う
-   UI・ゲームロジック・データを接続する
-   アプリの起動処理や画面遷移を管理する
-   動作確認・不具合調整を行う

------------------------------------------------------------------------

# 1. 歩数データの取得

このアプリでは歩数データがゲームの中心となるため、端末・統合担当はスマートフォンから歩数を取得する処理を実装する。

取得するデータ例

-   今日の歩数
-   過去数日の歩数
-   アプリ起動時の歩数更新

処理の流れ

    スマホの歩数データ
    ↓
    アプリで取得
    ↓
    ゲームロジックに渡す
    ↓
    UIに表示

------------------------------------------------------------------------

# 2. データ保存（セーブ・ロード）

ゲームでは、ユーザーの進行状況を保存する必要がある。

保存するデータ例

-   現在のレベル
-   現在の経験値
-   今日の歩数
-   連続ログイン日数
-   コイン数
-   前回起動日時

端末・統合担当はこれらを保存し、次回アプリ起動時に復元する。

------------------------------------------------------------------------

# 3. 各担当機能の統合

各担当が作成した機能を1つのアプリとして接続する。

例

    アプリ起動
    ↓
    保存データ読み込み
    ↓
    歩数取得
    ↓
    ゲームロジックで経験値計算
    ↓
    UIに結果表示
    ↓
    結果保存

------------------------------------------------------------------------

# 4. 画面遷移と起動処理

アプリ全体の動作フローを整理する。

例

-   起動時にホーム画面を表示
-   初回起動時のみ説明画面を表示
-   歩数取得に失敗した場合のエラー表示
-   保存データがない場合の初期化処理

------------------------------------------------------------------------

# 5. 動作確認とデバッグ

統合後には以下のような問題が発生する可能性がある。

例

-   歩数が反映されない
-   レベルは上がるが表示が更新されない
-   保存データが消える
-   画面遷移後にエラーが出る
-   端末によって挙動が異なる

これらを確認し修正する。

------------------------------------------------------------------------

# 他担当との違い

## UI担当

UI担当は **画面の見た目を作る**。

例

-   キャラクター表示
-   ボタン配置
-   レベル表示デザイン

端末・統合担当は **その画面を実際に動作させる**。

例

-   データをUIに渡す
-   ボタンの動作を接続する
-   画面遷移を実装する

------------------------------------------------------------------------

## ゲームロジック担当

ゲームロジック担当は **ゲームの計算ルールを作る**。

例

-   経験値計算
-   レベルアップ判定
-   報酬計算

端末・統合担当は **そのロジックをアプリの中で呼び出す**。

例

-   歩数をロジックに渡す
-   計算結果を画面に反映する
-   更新結果を保存する

------------------------------------------------------------------------

## データ担当

データ担当は **ゲーム設定データを作る**。

例

-   レベルテーブル
-   報酬テーブル
-   状態テーブル

端末・統合担当は **そのデータを読み込み、実際に利用する**。

------------------------------------------------------------------------

# 実装する機能

端末・統合担当が実装する機能例

-   歩数取得処理
-   データ保存処理
-   データ読み込み処理
-   起動時初期化
-   画面遷移処理
-   エラー処理

# 典型的な処理フロー

    1. スマートフォンから今日の歩数を取得
    2. 歩数をゲームロジックに渡す
    3. 経験値を計算
    4. レベルを更新
    5. UIに結果を表示
    6. 更新データを保存

------------------------------------------------------------------------

# まとめ

端末・統合担当は、

-   スマートフォンの機能を扱う
-   各担当の機能を接続する
-   アプリを実際に動く状態にする

という役割を担う。

この担当は、**アプリを完成させるための最終工程を担当する重要な役割**である。
